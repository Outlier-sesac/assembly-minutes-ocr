##### 국회 회의록 OCR 및 데이터 처리 파이프라인 분석 보고서

이 코드는 **국회 회의록 PDF 파일을 텍스트로 변환하고 구조화된 데이터로 가공하는 종합 처리 시스템**입니다.

**주요 처리 흐름:**

1. **PDF 전처리 및 OCR**: Google Drive에서 국회 회의록 PDF 파일을 로드하고, 2단 형식의 PDF를 좌우로 분할하여 Upstage OCR API를 통해 텍스트로 변환합니다.
2. **텍스트 분할 및 요약**: 변환된 텍스트를 발언자별로 분할하고, Solar-Pro 모델을 사용해 각 발언을 1-2문장으로 요약하여 JSON 형태로 저장합니다.
3. **데이터프레임 구축**: JSON 파일들을 읽어 회의록ID, 회의종류, 회의일자, 대수, 회기, 차수, 발언순번, 발언자, 발언요약 등의 구조화된 컬럼을 가진 Pandas DataFrame으로 변환합니다.
4. **발언자 정보 분리**: LLM을 활용해 '발언자' 필드에서 직책과 실제 발언자명을 분리하여 '직책'과 '분리된_발언자' 컬럼으로 나누어 처리합니다.
5. **비동기 처리 최적화**: API 호출 시 세마포어를 통한 동시 요청 수 제한, 재시도 로직, 백오프 전략을 적용하여 대용량 데이터 처리 성능을 향상시킵니다.
6. **데이터 병합 및 저장**: 원본 회의록 데이터와 분리된 발언자 정보를 병합하여 최종적으로 CSV 파일로 저장하는 완전한 ETL 파이프라인을 구현합니다.
7. **오류 처리 및 복구**: 파일명 정규화, JSON 파싱 오류 처리, API 제한 대응 등 다양한 예외 상황에 대한 견고한 처리 로직을 포함합니다.
